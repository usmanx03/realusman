How I think

Good backend work is invisible. The API responds fast, the data is consistent, the service recovers from failure without waking anyone up. That is what I build toward.

I think in systems: how components interact, where bottlenecks form, what breaks under load. If something is hard to debug in production, it is not finished yet.

I reach for boring technology that works rather than clever technology that surprises. One well-understood solution beats five that each handle a piece of the problem.
